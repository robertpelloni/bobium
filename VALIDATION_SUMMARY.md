This document synthesizes the completed C++ patches, orchestration logic, and validation requirements for the final local build phase of the bobium project.

## 2. Local Execution Guide

To validate and compile these patches on a high-compute physical machine:

1. **Provision Hardware:** Ensure a Linux/macOS/Windows build environment with >150GB disk space and 32GB+ RAM.
2. **Clone the Hub:** `git clone --recursive https://github.com/robertpelloni/bobium.git && cd bobium`
3. **Verify Host Checks:** `./scripts/pre_flight_check.sh`
4. **Fetch Upstream Source:** `./scripts/fetch-chromium.sh` (Downloads depot_tools and triggers `gclient sync`. Takes several hours.)
5. **Fetch Ungoogled Patches:** `./scripts/fetch-ungoogled.sh`
6. **Apply bobium Patches:** `./scripts/apply-patches.sh`
7. **Initiate Build:** `./scripts/build.sh` (Compiles the Chromium target via `gn gen` and `autoninja`. Takes 4-8 hours).
8. **Package Binaries:** `./scripts/package_release.sh`

## 3. Expected Build Artifacts & Checksums

Post-compilation, the following primary artifacts must exist in `chromium/out/Release/`:

| Artifact | Purpose | Example Validation |
|----------|---------|---------------------|
| `chrome` / `chrome.exe` | Main binary executable. | Runs and spawns UI. |
| `resources.pak` | Embedded UI and branding assets. | Custom bobium branding visible. |
| `.portable` (User created) | Triggers Vault mode. | Generates adjacent `bobium_profile/` |

*(Note: Exact SHASUM256 checksums will be generated by the `package_release.sh` step once the binary is compiled on the target architecture.)*
